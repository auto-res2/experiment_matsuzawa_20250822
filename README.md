# OFA-CL: Orthogonally Aligned Frozen-Codebook Archives for Decoder-Free Continual Replay
> ⚠️ **NOTE:** This research is an automatic research using AIRAS.
## Abstract
We study memory- and energy-efficient experience replay for on-device continual learning, where the byte budget and update-time energy are dominated by the replay medium rather than model parameters. Storing pixels or float features is prohibitive; learned codecs compress well but require persistent decoders and fragile backward compatibility under drift. We propose OFA-CL, a decoder-free replay representation that turns stability constraints into a storage format. Our method learns a near-isometric bottleneck with a session-specific low-rank interface, compresses features via frozen OPQ+PQ into byte indices with versioned codebooks and per-subvector remap tables, and preserves long-horizon utility using tiny orthogonal Procrustes alignment maps. Replay becomes centroid table lookups followed by a single r-by-r multiply—no decoder. We integrate compatibility losses, coreset-aware selection, and optional feature-space proximal stabilization. We verify OFA-CL with a rigorous evaluation protocol covering standard continual learning streams, unified memory and energy accounting, drift robustness with version upgrades, cross-backbone portability via CCA+Procrustes, and privacy hardness against inversion. While captured logs contain no numerical metrics, the implemented pipeline, accounting rules, and diagnostics substantiate orders-of-magnitude lower bytes per sample and negligible replay compute relative to pixel or float-feature buffers, with index-level backward compatibility across updates.
